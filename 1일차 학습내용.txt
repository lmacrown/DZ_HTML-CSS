정적 사이트 : 정보만을 보여주는 사이트
동적 사이트 : 정적 사이트+기능
서버 : 인터넷에 연결된 컴퓨터
클라이언트 : 사용자가 웹 사이트에 접근하려고 사용하는 기기
요청 정보 및 응답 자료 : html/htmls(암호화)형식
http(s) : text 기반 프로토콜
=> S : 보안
서버 : 웹 사이트에서 정보를 검색 및 처리
클라이언트 : 사용자와 서버 사이에서 화면으로 결과를 보여줌
프론트 엔드 : 사용자 앞에 보이는 영역. 디자인(css, html), js
- 영역 : 다양한 라이브러리, 프레임워크
백 엔드 : DB나 기능 담당. jsp, php, asp.net, 파이썬, spring boot 및 DB처리
- 리눅스 서버, 네트워크, 데이터베이스 구축 기술 및 서버언어, 프레임워크
브라우저->웹서버->WAS(tomcat)
DHCP : 서버로부터 IP를 받아옴

MVC : 모델, 뷰, 컨트롤러로 역할을 구분한 패턴.
MVC2 : 하나의 Servlet이 받아 그 결과를 JSP 페이지로 전달. 유지보수와 확장에 용이
Model : 상태 변화 시 Controller 와 View에 이를 알림
View : 사용자가 볼 결과물 생성
Controller : 모델의 상태 변경
옵저버 디자인 패턴 : 

HTML : 웹 브라우저 창에 웹 문서의 내용을 보여 주는 데 필요한 약속
CSS : HTML로 만든 내용을 사용자가 알아보기 쉽데 꾸미거나 사용하기 편리하도록 배치
JS : 사용자의 행동에 따라 반응하는 웹
